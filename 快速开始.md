# Excel 工具箱 - 快速开始指南

## 🎉 欢迎使用 Excel 工具箱！

应用已成功启动，后端已连接。现在可以开始测试所有功能了！

## 🚀 立即开始

### 1️⃣ 打开应用窗口
Electron 窗口应该已经自动打开。如果没有，请检查任务栏。

### 2️⃣ 检查连接状态
查看左侧边栏底部：
- ✅ **绿色圆点 + "后端已连接"** = 一切正常
- ❌ **灰色圆点 + "后端未连接"** = 需要重启应用

### 3️⃣ 加载第一个文件

**步骤**：
1. 点击左侧 **"📁 文件管理"** 菜单
2. 在输入框中粘贴测试文件路径：
   ```
   C:\Users\12607\Desktop\excel工具箱—kiro版\python-backend\tests\test_files\test_load.xlsx
   ```
3. 点击 **"📂 加载文件"** 按钮
4. 等待几秒，文件信息将显示出来

### 4️⃣ 尝试内容处理

**步骤**：
1. 点击左侧 **"✏️ 内容处理"** 菜单
2. 选择任意操作卡片，点击 **"执行操作"** 按钮
3. 观察进度条显示
4. 查看操作日志中的结果

### 5️⃣ 查看操作日志

**步骤**：
1. 点击左侧 **"📋 操作日志"** 菜单
2. 查看所有操作记录
3. 点击 **"清空日志"** 可以清除所有记录

## 💡 使用技巧

### 1. 文件路径输入
- 可以直接粘贴完整路径
- 支持 Windows 路径格式（如 `C:\Users\...`）
- 未来版本将支持文件选择对话框

### 2. 进度显示
- 长时间操作会显示进度覆盖层
- 进度条实时更新
- 显示当前处理的工作表

### 3. 错误提示
- 所有错误都会在操作日志中显示
- 红色标签表示错误
- 错误消息包含详细原因

### 4. 自动备份
- 每次保存前自动创建备份
- 备份文件保存在隐藏的 `.文件名_backups/` 目录
- 自动保留最近 5 个备份

### 5. 批量处理
- 内容处理操作默认处理所有工作表
- 进度条显示总体进度
- 结果显示成功/失败的工作表数量

## ❓ 常见问题

### Q: 后端未连接怎么办？
**A**: 
1. 检查控制台是否有错误信息
2. 确保 Python 已安装（`python --version`）
3. 确保依赖已安装（`pip list | findstr "openpyxl pandas"`）
4. 重启应用（关闭窗口，重新运行 `npm run dev`）

### Q: 文件加载失败怎么办？
**A**:
1. 检查文件路径是否正确
2. 检查文件是否存在
3. 检查文件是否被 Excel 占用（关闭 Excel）
4. 查看操作日志中的错误消息

### Q: 操作没有反应怎么办？
**A**:
1. 检查是否已加载文件
2. 查看操作日志中的错误消息
3. 检查控制台是否有错误信息
4. 尝试重新加载文件

### Q: 进度条一直不动怎么办？
**A**:
1. 等待几秒（大文件处理需要时间）
2. 检查控制台是否有错误信息
3. 如果超过 30 秒无响应，重启应用
